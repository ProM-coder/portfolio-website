Case Study

Budget Tracker App

An Android expense tracking application built to demonstrate CRUD functionality, SQLite persistence and practical mobile app logic.

Challenge

Users needed a simple way to record expenses, view existing records, update incorrect entries and delete expenses they no longer needed.

Approach

  • Designed a clean Android interface for adding and managing expenses.
  • Used Java for the application logic.
  • Used SQLite for local database persistence.
  • Implemented RecyclerView for displaying expenses.
  • Built create, read, update and delete functionality.

Outcome

The app supports full expense management and shows practical understanding of mobile development, database storage and user flow design.
